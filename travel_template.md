# 旅遊規劃書 資訊架構藍圖 (Information Architecture)
# 用途：定義規劃書四部分時間線與各章節產出重點。
# 經濟版 prompt 與 orchestrator 皆以此為統一藍圖，本檔不含任何 dynamic 標籤與 {變數} 佔位符。
# 各章節的實際檢索規則、公式與規範，一律以 prompt_generate_trip.md / prompt_update_trip.md / orchestrator/ 為準；
# 本檔只負責「章節順序」與「每章要交付什麼」的骨架共識。

## 第一部分：出發前準備篇（在出發地辦完）

1. **旅遊相關保險**
   對比 2~3 家台灣主流保險公司方案，聚焦突發疾病醫療額度、班機延誤起賠時數與額度。

2. **網路與漫遊方案**
   對比 eSIM / 實體 SIM / WiFi 分享器，並納入指定電信商的漫遊資費，給出 CP 值推薦。

3. **支付工具與匯率**
   說明主流支付工具生態與最新匯率，以 Step-by-Step 列點交代申辦、儲值、提領退費，並標示需備現金的場景。

4. **行李與違禁品規定**
   依指定航空公司查最新託運/手提限重與超重費率，並核對出發地與目的地民航局違禁品清單。

## 第二部分：機場通關篇（當天照著跑）

5. **出發機場 ➔ 目的機場 整合通關流程**
   從出發前線上申報到降落後入境查驗，整合成一條 Step-by-Step 流程，並標示每一步要出示什麼（護照、QR Code 等）。

## 第三部分：抵達在地篇（隨時翻閱）

6. **機場 ➔ 市區接駁交通對比**
   統整所有接駁方案（快線、巴士、計程車、包車），列出票價、班次、購票方式，並附同伴友善度評比。

7. **免稅與退稅**
   說明起徵門檻與退稅流程（店家即時退稅、機場退稅、App 退稅）。

8. **當地習俗與注意事項**
   社交禁忌、小費文化、交通禮儀、近期氣候與穿搭提醒。

9. **推薦 App**
   須與前文交通、支付工具呼應對應，避免推薦互不相關的工具。

10. **景點與美食行程規劃**
    嚴格依 prompt 中「行程與美食推導規則」（五大公式 A–F）產出，每半日行程含景點、同伴友善度、美食（依美食偏好雙軌 + 多元約束）、歇腳處、下一站交通。

## 第四部分：緊急應變篇（預防萬一）

11. **緊急情況與聯絡資訊**
    當地報警/消防/救護熱線、駐外館處電話地址、鄰近可英文/中文就醫的醫院資訊。
